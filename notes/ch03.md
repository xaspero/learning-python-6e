### Modules

A module is a 'package' of variable names, which are collectively known as 'namespace'; and the names within that package are called 'attributes' of the module. Attributes are attached to a specific objects, e.g. a module:

### import and from

_from_ copies modules' attributes, and makes those attributes variables in the recipient code (program), such that you can use the variables straightaway:

```
    from script import title
    print(title)
```

_import_ statement loads the module as a whole, and then you can use the module's name to qualify its attributes:

```
    import script
    print(script.title)

```

import runs a file only once per session, because import is a process - a program run. This means later import does nothing. The REPL must be stopped and restarted for import to be repeated.

_reload_ function is called to force Python to run an already imported file again in the same session without stopping and restarting the REPL. The _reload_ function lives inside the **importlib** standard library, and as such, it must be imported to be runned on an already imported module:

```
    fromimportlib import reload
    reload(script) # reloads the module again in the same session
```
